🌧 Люблю грозу в начале мая

ТЗ на разработку трекера дождливых занятий
мини-сайт / PWA-виджет · без базы данных · офлайн-режим

1. Общая концепция

Трекер привычек/заданий, приуроченных к дождливой погоде. Пользователь выбирает количество заданий (12 / 22 / 30), последовательно выполняет их в дождливые дни, отмечает как выполненные одним касанием. Приложение работает как PWA — добавляется на главный экран смартфона в виде большого виджета, не требует регистрации или сервера.

📌 Вся работа с данными — только localStorage браузера. Никакой БД, никакого бэкенда.

3. Дизайн-система

Цветовая палитра

Молочный фон #F5F0E8
Чёрный контур #1A1A1A
Жёлтый #FFD54F
Голубой #90CAF9
Красный/розовый #EF9A9A

Типографика

  • Заголовок приложения — Caveat Bold 28–32px
  • Названия заданий — Caveat SemiBold 18px
  • Текст карточек — Caveat Regular 15–16px
  • Счётчик прогресса — Caveat Bold 20px

Визуальный стиль

  • Все элементы — как нарисованные от руки: слегка неровные края, SVG с имитацией карандашной линии
  • Зачёркивание выполненного — обычная чёрная линия поверх иконки (SVG line)
  • Карточки — белый прямоугольник с чёрным контуром 2px, скруглённые углы 12px
  • Фон всего приложения — молочный #F5F0E8
  • Никаких теней, градиентов, глянца — только плоский рисованный стиль

4. Структура экранов

Экран 1 — Приветствие / выбор режима

Заголовок «Люблю грозу в начале мая»
→
Выбор: 12 / 22 / 30 заданий
→
Кнопка «Начать»

Экран 2 — Главный трекер (виджет)

  • Сверху: надпись «Люблю грозу в начале мая» — крупно, рукописным шрифтом
  • Черта-разделитель (нарисованная, немного кривая)
  • Счётчик прогресса: «выполнено X из N» + маленькая дождевая капля рядом
  • Сетка карточек-иконок: по 3–4 в ряд, адаптивно
  • Каждая карточка — SVG-рисунок + название задания под ним
  • Выполненные карточки перечёркнуты чёрной линией

Экран 3 — Раскрытая карточка (при удерживании)

  • Затемнение фона (overlay)
  • Крупная карточка по центру: SVG-иллюстрация + название + текст задания полностью
  • Кнопка закрыть «×»
  • Подсказка: «коснись рисунка — отметить выполненным»

5. Логика взаимодействия

ЖестДействие
Короткое касание карточкиЗачёркивает иконку + сохраняет в localStorage
Удержание карточки (500мс+)Открывает полную карточку с текстом задания
Касание рисунка в открытой карточкеОтмечает задание выполненным, закрывает карточку
Касание затемнения или «×»Закрывает карточку без отметки
Повторное касание выполненногоСнимает отметку (отмена)
⚡ Тап vs удержание: разделяются через touchstart + setTimeout(500ms). Если за 500мс не было touchend — открывается карточка. Если touchend произошёл раньше — это тап (зачёркивание).

6. PWA-конфигурация (виджет на главном экране)

  • manifest.json: name, short_name, icons (192×192 и 512×512), display: "standalone", background_color: "#F5F0E8", theme_color: "#1a1a1a", start_url: "/"
  • Service Worker: кешировать index.html + шрифты + SVG-иконки → работа офлайн
  • Иконки приложения: SVG с жёлтым дождевиком на молочном фоне, экспортировать в PNG 192 и 512px
  • display: "standalone" — скрывает браузерный UI, выглядит как нативное приложение
  • Минимальная высота — вся страница 100svh, контент не скроллится при выборе «12 заданий»

7. Хранение данных (localStorage)

Структура JSON-объекта в ключе rainTracker:

{ "mode": 12, "completed": [1, 4, 7], "startedAt": "2025-05-01" }
  • mode — выбранный режим (12 / 22 / 30)
  • completed — массив ID выполненных заданий
  • startedAt — дата начала (для статистики)

10. Ключевые требования к реализации

  • Работает без интернета после первой загрузки (Service Worker)
  • Сохраняет прогресс в localStorage — данные не теряются при закрытии
  • Адаптив: корректно отображается на экранах 320px–430px (все современные смартфоны)
  • Нет зависимостей от внешних API кроме Google Fonts
  • Весь JS — vanilla, без фреймворков
  • SVG-иллюстрации встроены inline в HTML
  • При полном выполнении — анимация/экран финала («Ты пережил(а) всю непогоду!»)
  • Кнопка сброса прогресса (скрытая, например 5 тапов по заголовку)
